### 项目简介
为用户提供了商家信息查询、秒杀优惠券等功能，同时帮助商家推广优惠信息。

### 技术栈
SpringBoot + MySQL + Redis + Lua + Kafka + Caffeine + Mybatis

### 主要工作
- **秒杀防超卖和一人一单**：使用 Redis 存储库存和订单信息，Lua 判断用户下单资格，实现库存不超卖、一人一单；
- **秒杀流程优化**：修改同步流程，使用消息队列异步处理库存扣减和订单生成，提高秒杀场景的并发性能；
- **未支付订单到期自动关闭**：使用 Spring Task 定时任务实现未支付订单的到期自动关闭；
- **支付和关单的并发**：使用乐观锁解决订单支付和关单的并发问题；
- **缓存优化**：使用逻辑过期方案防止 Redis 热点 Key 的缓存击穿问题，使用缓存空值方案解决 Redis Key 缓存穿透问题；
- **数据一致性保证**：更新数据库后删除缓存，若删除失败消息队列补偿重试，TTL 兜底共同保证数据一致性；
- **多级缓存**：使用 Caffeine 本地缓存和 Redis 缓存搭建二级缓存架构，提高热点数据访问速度，降低 Redis 压力。
